# 🛰️ SensorMeteo 

**SensorMeteo** — это профессиональная система мониторинга климата на базе Raspberry Pi 5 и Wemos D1 Mini. Проект объединяет локальные замеры с датчиков BME280/680 и внешние данные Open-Meteo в едином Telegram-интерфейсе.

---

## 📦 Python Библиотеки (Dependencies)

Для работы серверной части на Raspberry Pi требуются следующие пакеты:


| Библиотека | Назначение |
| :--- | :--- |
| `pyTelegramBotAPI` | Взаимодействие с Telegram Bot API |
| `Flask` | Прием HTTP POST данных от датчиков (порт 25580) |
| `requests` | HTTP-запросы к API погоды |
| `PySocks` | Поддержка SOCKS5 прокси (для работы через Xray) |
| `openmeteo-requests` | Клиент для работы с метеоданными |
| `pandas` | Обработка временных рядов и массивов данных |
| `requests-cache` | Кэширование ответов API погоды |

### Быстрая установка:
```bash
pip install pyTelegramBotAPI Flask requests pandas openmeteo-requests requests-cache retry-requests PySocks
```

---

## 🏗 Архитектура и Сеть

Система спроектирована для работы в сложных сетевых условиях:
*   **Входящий трафик (FRP)**: Проброс портов `25580` (данные датчиков) и `25569` (SSH/управление) через внешний VPS.
*   **Исходящий трафик (Xray)**: Бот общается с Telegram через локальный SOCKS5 мост (`127.0.0.1:25590`), используя протокол **VLESS Reality gRPC**.
*   **База данных**: SQLite используется для хранения истории замеров и настроек пользователей.

## 🛠 Установка

1. **Клонируйте репозиторий:**
   ```bash
   git clone [https://github.com/An10An10/SensorMeteo.git]
   cd SensorMeteo
   ```

2. **Настройте конфиг:**
   Укажите ваш API токен в `main.py` и настройте прокси:
   ```python
   apihelper.proxy = {'https': 'socks5h://127.0.0.1:25590'}
   ```

3. **Запустите службы:**
   Рекомендуется использовать `systemd` для автозапуска бота и Xray.

## 🤖 Команды бота
*   `/start` — Регистрация и первичный ввод координат (широта, долгота).
*   `/mysens` — Данные с ваших датчиков (t°, влажность, давление).
*   `/getweather` — Интерактивный прогноз погоды с выбором времени.
*   `/setregion` — Изменение местоположения для прогноза.
*   `/addsens` — Добавление нового датчика в личный список по ID.
